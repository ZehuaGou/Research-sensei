# 08_REVIEW_CHECKLIST

每天早上检查 Agent 夜间工作时，只看这几个点：

## 1. 文件变更

```bash
git status
git diff --stat
git diff
```

检查：

- 有没有大批无意义空文件？
- 有没有改不该改的文件？
- 有没有删除已有代码？
- 有没有把 key 写进代码？

## 2. 测试

```bash
pytest -q
```

检查：

- 测试是否真的存在？
- 测试是否只是空断言？
- 失败有没有被记录？

## 3. 当前阶段是否越界

对照 `03_FULL_IMPLEMENTATION_PLAN.md`：

- 它是否只做了当前 Phase？
- 是否提前做了后面复杂功能？
- 是否把未来功能写成假实现？

## 4. 输出是否有证据意识

检查 JSON/card：

- 是否有 evidence_status？
- 是否有 `UNVERIFIED` / `NEEDS_HUMAN_CHECK`？
- 是否把推测写成事实？

## 5. 页面是否可用

如果做到了 render：

- 页面是否能打开？
- 字体是否可读？
- 卡片是否挤压？
- 公式是否渲染？
- 追问入口是否存在？

## 6. 继续/回滚决策

- 如果测试通过、文件改动合理：继续下一阶段。
- 如果测试通过但功能空：要求补实现。
- 如果测试失败：先修测试，不推进。
- 如果文件乱改：回滚后重新给更小任务。
