# Phase 4 - 单篇论文输入与解析

## 目标

本地 markdown/txt/PDF 输入转 sections/blocks。

## 执行原则

- 只做本阶段任务。
- 每个任务必须有测试或可验证命令。
- 测试失败不进入下一任务。
- 不确定写入 `OPEN_QUESTIONS.md`。

## 输入文档

- `01_ARCHITECTURE.md`
- `02_MODULE_CONTRACTS.md`
- `03_FULL_IMPLEMENTATION_PLAN.md`
- `04_ACCEPTANCE_CRITERIA.md`
- `05_TEST_PLAN.md`
- `06_AGENT_RULES.md`

## 阶段任务

请在 `03_FULL_IMPLEMENTATION_PLAN.md` 中找到对应 Phase 的所有 Task，逐个执行。

## 阶段完成后必须输出

```text
Phase 完成报告：
- 完成任务：
- 修改文件：
- 新增测试：
- 测试命令：
- 测试结果：
- 未完成/降级：
- 需要人工确认：
- 是否允许进入下一 Phase：是/否
```

## 阶段停止条件

- 测试失败且修复一次仍失败。
- 需要用户确认技术选择。
- 需要真实 API key 或联网。
- 发现已存在代码和本阶段设计冲突。
